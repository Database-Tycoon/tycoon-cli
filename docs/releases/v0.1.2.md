## Candidates carried forward from v0.1.1 "What's next"

These were called out as v0.1.2 candidates when v0.1.1 shipped. Triage
and promote into the "Added"/"Changed" sections above as work starts.

- **Cloud-warehouse alignment**: the same warehouse-alignment check we
  now do for DuckDB, extended to MotherDuck / Snowflake / BigQuery
  connection strings. When a registered dbt project targets a
  different cloud warehouse than tycoon's, warn and offer to adopt.
- **`tycoon register warehouse`** and **`tycoon register ingestion`**:
  extend the `register` family to cover the other two stack components
  that don't currently have one. Shape mirrors `tycoon register dbt/rill`.
- **Network-gated end-to-end tests in CI**: one test per template that
  runs a full `init → sources run → analyze → build` cycle against real
  external sources. Mark `@pytest.mark.e2e` and run only on a labelled
  CI job so the main suite stays fast and offline.
